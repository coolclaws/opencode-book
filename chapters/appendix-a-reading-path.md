# 附录 A：推荐阅读路径

> "一本好书应该像一座城市——你可以从任何一个入口进入，但总有几条路线让你看到最精彩的风景。"

本书共 22 章，覆盖了 OpenCode 从基础使用到架构深度分析的方方面面。根据读者的背景和目标，我们推荐以下三条阅读路径。每条路径都经过精心设计，确保前置知识在需要时已经具备。

## 路径一：快速上手

**目标读者**：希望尽快使用 OpenCode 提升开发效率的工程师

**预计时间**：2-3 小时

**前置要求**：熟悉命令行操作，有基本的 TypeScript 或 JavaScript 经验

| 顺序 | 章节 | 内容 | 你将学到 |
|------|------|------|---------|
| 1 | 第 1 章 | 导论：为什么要读 OpenCode 源码 | 项目定位、核心优势 |
| 2 | 第 3 章 | 启动流程 | 从命令行到服务就绪 |
| 3 | 第 7 章 | Tool 系统 | 内置工具的能力和用法 |
| 4 | 第 8 章 | 权限与安全 | 安全模型与权限配置 |

**学习成果**：理解 OpenCode 的核心工作流，掌握工具使用和权限配置，能够在日常开发中有效使用 OpenCode。

**后续建议**：阅读第 22 章（配置体系）深入自定义配置，或阅读第 21 章（插件系统）扩展功能。

**实践检验**：完成路径一后，你应该能够回答以下问题：

- OpenCode 启动时经历了哪几个阶段？
- `Tool.define()` 的参数结构是什么？
- 权限的三个级别（`allow`、`ask`、`deny`）分别在什么场景下使用？
- 如何通过配置文件自定义工具权限？

## 路径二：架构理解

**目标读者**：希望理解 AI Coding Assistant 设计原理的架构师和技术负责人

**预计时间**：8-10 小时

**前置要求**：有系统设计经验，了解事件驱动架构和 ORM 概念

| 顺序 | 章节 | 内容 | 你将学到 |
|------|------|------|---------|
| 1 | 第 1 章 | 导论 | 项目全貌与设计理念 |
| 2 | 第 2 章 | 技术选型 | Bun + Hono + SolidJS + Drizzle 的选型逻辑 |
| 3 | 第 4 章 | 服务端架构 | Hono 路由、API 设计 |
| 4 | 第 5 章 | 存储层 | SQLite + Drizzle ORM 数据模型 |
| 5 | 第 6 章 | 事件总线 | Bus 系统与事件驱动架构 |
| 6 | 第 9 章 | Provider 层 | 多模型适配与 AI SDK 集成 |
| 7 | 第 14 章 | Session 管理 | 会话生命周期与上下文管理 |
| 8 | 第 15 章 | 上下文压缩 | Compaction 与 Token 管理 |

这条路径的核心思路是 **自底向上**：先理解基础设施层（服务端、存储、事件），再理解业务层（Provider、Session），最后理解智能层（Compaction）。这种顺序与 OpenCode 源码中模块的依赖关系一致——`provider` 依赖 `config`，`session` 依赖 `storage` 和 `bus`，`compaction` 依赖 `session` 和 `provider`。

**学习成果**：深入理解 AI Coding Assistant 的系统架构，包括多模型适配、上下文管理、事件驱动等核心设计模式。

**后续建议**：根据兴趣选读——对 MCP 感兴趣读第 10 章，对 LSP 感兴趣读第 11 章，对 Agent 设计感兴趣读第 13 章。

**实践检验**：完成路径二后，尝试画出 OpenCode 的模块依赖图，标注每个模块的职责和数据流向。如果你能独立画出包含 Server → Bus → Session → Provider → Tool 的调用链路，说明你已经掌握了核心架构。

## 路径三：完整精读

**目标读者**：希望全面掌握 OpenCode 源码、准备贡献代码或构建类似系统的开发者

**预计时间**：20-30 小时

**前置要求**：扎实的 TypeScript 基础，熟悉 Zod、Effect 等库，有开源项目贡献经验者更佳

| 阶段 | 章节 | 主题 |
|------|------|------|
| **基础篇** | 第 1-3 章 | 导论、技术选型、启动流程 |
| **核心篇** | 第 4-8 章 | 服务端、存储、事件、工具、权限 |
| **集成篇** | 第 9-12 章 | Provider、MCP、LSP、格式化 |
| **智能篇** | 第 13-16 章 | Agent、Session、压缩、Snapshot |
| **交互篇** | 第 17-20 章 | CLI、Worktree、TUI、Desktop/Web |
| **生态篇** | 第 21-22 章 | 插件系统、配置与部署 |
| **附录** | A-C | 阅读路径、工具速查、术语表 |

**学习建议**：

1. **边读边跑代码**：克隆 OpenCode 仓库，每章对照源码阅读。建议在 IDE 中打开 `packages/opencode/src/` 目录，利用 TypeScript 的类型跳转功能追踪调用链
2. **关注对比表格**：各章末尾与 Claude Code / Cursor 的对比有助于理解设计取舍
3. **动手实验**：每章的"实战"小节提供了可操作的练习
4. **利用附录 B**：工具速查表是日常使用的便捷参考，特别是在阅读第 7 章（Tool 系统）时可以交叉查阅
5. **利用附录 C**：术语表帮助理解陌生概念，遇到不确定的缩写（如 ACP、MCP）时随时翻阅
6. **记录问题**：每阶段结束后，记下尚未解决的疑问，往往后续章节会给出答案

**理解依赖图的技巧**：这张图的结构与 OpenCode 源码中 `packages/opencode/src/` 目录下的模块依赖关系高度对应。例如，`tool/` 目录下的工具实现依赖 `permission/` 的权限检查，正如第 7 章依赖第 8 章的知识。`session/` 依赖 `storage/` 和 `bus/`，正如第 14 章建立在第 5 章和第 6 章的基础之上。当你在源码中看到 `import` 语句时，可以回到这张图来定位对应的章节。

## 常见问题

**Q：我只对 MCP 协议感兴趣，可以直接跳到第 10 章吗？**

可以。第 10 章的前置知识主要来自第 9 章（Provider 层），因为 MCP 服务器在 OpenCode 中是作为外部 Provider 的扩展机制存在的。建议至少快速浏览第 9 章的 Provider 注册流程。

**Q：我想基于 OpenCode 的架构构建自己的 AI 编程助手，应该重点读哪些章节？**

建议按路径三完整精读，但优先级排序为：第 2 章（技术选型决策）→ 第 7-8 章（Tool + 权限，这是区别于普通 chatbot 的核心）→ 第 13-15 章（Agent 循环 + Session + Compaction，这是 AI 编程助手的灵魂）→ 第 9 章（多 Provider 适配）。

**Q：本书的代码示例基于哪个版本的 OpenCode？**

本书基于 OpenCode 开源后的早期稳定版本。由于项目仍在活跃开发中，部分 API 签名可能已发生变化，但核心架构模式保持稳定。建议对照最新源码阅读。
